    beginsWith

    • beginsWith(
          sort: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>,
          options?: QueryOptions<T, PK, SK, keyof T> & { select?: undefined },
      ): Promise<QueryResult<T>>

      Fetch records whose sort key begins with the given value.

      Passing a partial object like { status: 'draft' } only populates the sort-key fields you provide — the remaining fields are omitted from the composite key, so the generated prefix matches every record that shares the leading portion.

      Parameters

      • sort: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>

        Object of sort-key field values, or a raw string prefix if you need to bypass key construction.

      • Optionaloptions: QueryOptions<T, PK, SK, keyof T> & { select?: undefined }

        Optional QueryOptions.

      Returns Promise<QueryResult<T>>

      QueryResult.

      Example

      // Every post whose postTitle starts with "aa"
      const aa = await PostFacet.GSIPostByTitle
        .query({ pageId: 'p1' })
        .beginsWith({ postTitle: 'aa' });

      Example

      Projected, requires pickValidator on the facet:

      const aa = await PostFacet.GSIPostByTitle
        .query({ pageId: 'p1' })
        .beginsWith({ postTitle: 'aa' }, { select: ['postStatus'] });

    between

    • between(
          start: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>,
          end: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>,
          options?: QueryOptions<T, PK, SK, keyof T> & { select?: undefined },
      ): Promise<QueryResult<T>>

      Fetch records whose sort key is between start and end, inclusive on both ends.

      Composite sort keys are compared lexicographically in their string-joined form. For date-valued sort keys that ISO-encode, a calendar range "between Jan 1 and Feb 28" works as expected.

      Parameters

      • start: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>

        Object of sort-key field values for the lower bound, or a raw string.

      • end: string | Partial<Pick<T, ActiveSK<SK, GSIPK, GSISK>>>

        Object of sort-key field values for the upper bound, or a raw string.

      • Optionaloptions: QueryOptions<T, PK, SK, keyof T> & { select?: undefined }

        Optional QueryOptions.

      Returns Promise<QueryResult<T>>

      QueryResult.

      Example

      const range = await PostFacet.GSIPostByTitle
        .query({ pageId: 'p1' })
        .between({ postTitle: 'ab' }, { postTitle: 'ae' });

      Example

      Projected, requires pickValidator on the facet:

      const range = await PostFacet.GSIPostByTitle
        .query({ pageId: 'p1' })
        .between(
          { postTitle: 'ab' },
          { postTitle: 'ae' },
          { select: ['postStatus'] },
        );

    first

    • first(
          options?: Omit<QueryOptions<T, PK, SK, keyof T>, "cursor" | "limit"> & {
              select?: undefined;
          },
      ): Promise<T | null>

      Fetch the first record in the partition (or null if empty).

      Internally runs list({ limit: 1 }) and unwraps the first row. Useful for "does any record exist for this partition?" checks and for cheap "earliest/latest" lookups when combined with scanForward.

      Parameters

      • Optionaloptions: Omit<QueryOptions<T, PK, SK, keyof T>, "cursor" | "limit"> & {
            select?: undefined;
        }

        Subset of QueryOptions; cursor and limit are not meaningful here and are omitted.

      Returns Promise<T | null>

      The first record in the partition or null.

      Example

      // Most recent post (sort descending, take first)
      const latest = await PostFacet.GSIStatusSendAt
        .query({ postStatus: 'sent' })
        .first({ scanForward: false });

      Example

      Projected, requires pickValidator on the facet:

      const head = await PostFacet
        .query({ pageId: 'p1' })
        .first({ select: ['postTitle'] });
